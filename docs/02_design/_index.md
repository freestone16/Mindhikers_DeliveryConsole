## 🏗️ 模块导览

### [一级火箭: 黄金坩埚 (Crucible)](./crucible/_master.md) 🔥
专注人机高密度推演、脱水结晶化的思维草稿与大纲提炼。

### [二级火箭: 交付控制台 (Delivery Console)](./delivery/_master.md) 🏭
核心重兵把守处！专家矩阵（写作、营销、导演等）的调度，以及本地强隔离下的 Render Farm 生成流（文案、排版、长短视频、图片）。

### [三级火箭: 分发控制台 (Distribution Console)](./distribution/_master.md) 📡
面向全域多平台的项目组合收尾并进行 API 推流推送的模块管理。

### [信息深潜 (Info Hub)](./infohub/_master.md) 📻
雷达侦搜与素材提取层（P0）。采集如 YouTube、Twitter 等并转化为可用图文 markdown 基座。

### [基础设施 (Infrastructure)](./infrastructure/_master.md) ⚙️
项目级隔离管线、全局大模型、外部云与密钥的安全调度配置、网络状态映射。

## 📜 协同规则约定

1. **增删查改**：要了解任何一个特型功能（如：排版引擎如何渲染），进入对应模块子目录后，查看 `_master.md` 中第三节的关联索引获取对应的 `sd###` 详细子文档。
2. **新增资产**：新增设一个 API 设计或新加一个子系统，务必在对应模块的 `_master.md` 索引表中增加一行条目。并将其作为模块变更日志写在最下方。禁止新增不被任何 `_master.md` 收录的孤儿 Markdown 文件！
3. **老杨 (CodingMaster) 的天条**：进行任何功能的重构或者增加，自动在 `/save-progress` 与最终汇报前检查并同步以上状态机到最新！
