## 1. 背景与问题定义

Phase 2 的"渲染 AI 条目"按钮（`handleRenderChecked`）原来是 **纯 Mock 实现**：

```typescript
// ❌ 旧实现（stub）
fs.writeFileSync(outputPath, 'Mock video content');
```

用户点击后，后端立刻伪造 `.mp4` 文件，没有触发任何真实渲染。这导致：
- 用户期望等待 5-8 分钟的 Remotion 渲染 → 实际 0 秒假完成
- Volcengine 文生视频（火山引擎）同样被 Mock 跳过
- 前端没有自动轮询机制，用户必须手动点每张卡片的"生成预览"

## 2. 整体架构设计

### 2.1 流程图

```
用户点击"渲染 AI 条目"
        ↓
DirectorSection.handleRenderChecked
 ├─ 收集所有 isChecked & renderable 的 option（含完整 template/props/prompt）
 └─ POST /api/director/phase2/render_checked
            ↓
     server: phase2RenderChecked
      ├─ 1. 分类：Remotion队列 / Volcengine队列
      ├─ 2. 预注册所有任务为 'pending' → thumbnailTasks Map
      ├─ 3. 立即 res.json({ taskKeys: [...] })  ← 前端立刻得到key列表
      └─ 4. 后台并行启动两条流水线（Promise.all）
              ├─ Remotion队列 → 3-worker并发队列
              └─ Volcengine队列 → Promise.all全并发

前端收到 taskKeys
        ↓
setPendingBatchTaskKeys(new Set(taskKeys))
        ↓
pendingTaskKeys 向下传递：
Phase2View → ChapterCard → OptionRow
        ↓
OptionRow useEffect [isPendingBatch]：
  if (isPendingBatch && thumbStatus === 'idle') {
    pollThumbnail(taskKey)   // 每2秒轮询 /api/director/phase2/thumbnail/:key
  }
```

### 2.2 并发控制策略

| 引擎 | 并发策略 | 理由 |
|---|---|---|
| **Remotion** (本地) | 3 worker 共享 index | 本机 CPU 限制，多进程竞争会导致OOM |
| **Volcengine** (云端) | Promise.all 全并发 | 云端按需扩容，本地无需排队 |

## 3. 后端实现细节

### 3.1 Remotion 并发队列（Shared-Index Worker Pattern）

```typescript
const REMOTION_CONCURRENCY = 3;
const runRemotionQueue = async () => {
  let idx = 0;  // 共享游标（无锁，JS 单线程安全）
  const worker = async () => {
    while (idx < remotionQueue.length) {
      const chapter = remotionQueue[idx++];  // 原子取号
      const { chapterId, option } = chapter;
      const taskKey = `${chapterId}-${option.id}`;
      try {
        thumbnailTasks.set(taskKey, { status: 'processing', createdAt: Date.now() });
        const imageUrl = await buildRemotionPreview(option, chapterId, projectId);
        thumbnailTasks.set(taskKey, { status: 'completed', imageUrl, createdAt: Date.now() });
      } catch (err) {
        thumbnailTasks.set(taskKey, { status: 'failed', error: String(err), createdAt: Date.now() });
      }
    }
  };
  await Promise.all(Array.from({ length: Math.min(REMOTION_CONCURRENCY, remotionQueue.length) }, () => worker()));
};
```

### 3.2 Volcengine 队列

```typescript
const runVolcQueue = async () => {
  await Promise.all(volcQueue.map(async ({ chapterId, option }) => {
    const taskKey = `${chapterId}-${option.id}`;
    try {
      thumbnailTasks.set(taskKey, { status: 'processing', createdAt: Date.now() });
      const imageUrl = await generateImageWithVolc(option.imagePrompt || option.prompt);
      thumbnailTasks.set(taskKey, { status: 'completed', imageUrl, createdAt: Date.now() });
    } catch (err) {
      thumbnailTasks.set(taskKey, { status: 'failed', error: String(err), createdAt: Date.now() });
    }
  }));
};
```

### 3.3 taskKey 命名规范

```
taskKey = `${chapterId}-${optionId}`
```

同样的命名在：
- 后端 `thumbnailTasks` Map 的 key
- `GET /api/director/phase2/thumbnail/:taskKey` 的 URL 参数
- 前端 `OptionRow` 的 `const taskKey = \`${chapter.chapterId}-${option.id}\``

**三处必须保持一致。**

## 4. 前端实现细节

### 4.1 Prop 传递链

```
DirectorSection
  state: pendingBatchTaskKeys: Set<string>
  ↓ prop: pendingTaskKeys={pendingBatchTaskKeys}
Phase2View
  ↓ prop: pendingTaskKeys={pendingTaskKeys}
ChapterCard
  ↓ prop: isPendingBatch={pendingTaskKeys?.has(`${chapter.chapterId}-${option.id}`) ?? false}
OptionRow
  useEffect([isPendingBatch]) → pollThumbnail(taskKey)
```

### 4.2 OptionRow 自动轮询逻辑

```typescript
// 批量渲染触发后自动开始轮询
useEffect(() => {
  if (isPendingBatch && thumbStatus === 'idle') {
    console.log('[BatchPoll] Auto-starting poll for:', taskKey);
    pollThumbnail(taskKey);  // 每2秒轮询一次，completed/failed 自动停止
  }
}, [isPendingBatch]);
```

**设计要点**：
- 依赖 `[isPendingBatch]`（不是 `[isPendingBatch, thumbStatus]`）：避免状态循环
- `thumbStatus === 'idle'` 的守卫：防止重复触发已在运行的轮询
- 轮询函数本身已有终止逻辑（`completed` / `failed` 时不再 `setTimeout`）

## 5. 修复的 Bug 清单

| Bug | 原因 | 修复方式 |
|---|---|---|
| 批量渲染是 Mock | `phase2RenderChecked` 是 stub | 完整重写，实现真实并发队列 |
