## 2. 核心功能盘点 (The Control Deck)

### 2.1 编组折叠 (Job Groups)
从“提稿机 (Composer)”点那一次`[🚀 确认起飞]`，就会生成一个唯一的**任务组 (Job Group)**。无论你选了多少个平台，所有原子发布任务都会被折叠在这个组下面。老卢不需要在几百条流水里找对应关系。

### 2.2 错峰漏斗的终极反馈 (The Safety Valve)
当你看到 `🟡 状态: ⏳ 安全错峰倒计时` 时，这是我们在设计上对防风控底线的体现。但系统依然让渡了**最高控制权**给你：
- **[强制插队 ⚡]**：你如果特别急着上线，可以点击此按钮，立刻绕开延时缓冲发包。
- **[终止 ⏹️]**：发现上一个平台发出的文字其实有错别字，趁此时后面的平台还在“错峰睡眠”，立刻一键拦截中止！

### 2.3 灾难恢复与清创 (Disaster Recovery)
开源自动化脚本 (Playwright) 以及各家 API 的不稳定性是常态。
当出现 `🔴 状态: ❌ 失败` 时：
1. **[查看详细日志]**：点击后不跳出，原地展开一个终端黑框代码区，吐出纯正的 Node.js 错误堆栈，或者是抓取到的网页报错图片——**这让你去求助 GLM-5 排错时有图有真相。**
2. **[重新入队 🔄]**：排错后（比如去 Accounts Hub 补扫了风控码），回来一键 Retry 这个平台。绝对不要为了补发一个平台，跑去图文系统重来一遍，奥卡姆剃刀不允许这种浪费。
