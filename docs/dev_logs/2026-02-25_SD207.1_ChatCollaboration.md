## ❌ 关键问题：projectId 不一致

### 现象
- 前端显示当前项目: **CSET-SP3**
- 脚本数据保存到: **CSET-Seedance2**

### 根因分析
1. `server/shorts.ts` 中 `getProjectRoot()` 使用:
   ```typescript
   const PROJECTS_BASE = process.env.PROJECTS_BASE || path.join(process.cwd(), 'Projects');
   ```
2. `.env` 中 `PROJECTS_BASE=/Users/luzhoua/.../Projects` (正确)
3. 但 **`process.cwd()` 返回的是 DeliveryConsole 目录**，不是环境变量
4. 前端发送的 `projectId` 可能为空或错误

### 需要修复
1. 检查前端 `state.projectId` 的值
2. 确保 `server/shorts.ts` 使用环境变量中的 PROJECTS_BASE
3. 统一所有模块的 getProjectRoot 实现
