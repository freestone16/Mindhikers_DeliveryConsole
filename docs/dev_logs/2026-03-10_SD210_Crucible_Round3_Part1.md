## 本轮目标

围绕黄金坩埚第三轮方案，先只落“议题锁定工作台”这一条最小可提交切片，避免继续把上下文和实现边界摊大。

本轮切片包括：

1. 右侧 chatbox 降级为状态窗
2. 中区生成 3 个议题澄清问题卡
3. 支持逐题保存
4. 支持全部提交与二次确认
5. Header 右上项目 / 文稿在坩埚锁题阶段置灰

## 已完成

### 1. 右侧 chat 状态窗化

- 黄金坩埚模式下，assistant 长文本不再在右侧完整展开。
- chat 当前主要承担：
  - 欢迎 / 分析中提示
  - “主要问题已生成，请在中区逐项回答”
  - 金句 / 结构 / 参考材料已分流的状态提醒
- 右下输入框保留，但定位已降为“短追问入口”，不再承担长篇写作。

## 2. 中区议题锁定工作台成立

- 中区新增“议题锁定阶段”工作区。
- 系统当前每轮固定生成 3 个问题卡，围绕：
  - 核心判断
  - 最强反方 / 阻力
  - 读者应带走的新判断
- 每张卡已具备：
  - 问题标题
  - 辅助说明
  - 大反馈框
  - 保存按钮
  - 已保存 / 未保存状态

### 3. 提交机制首版

- 中区右上已加入“全部提交”按钮。
- 提交前会进行二次确认。
- 当前提交后先只记录“本轮已提交”状态，不做下一轮实际生成。

### 4. Header 锁定策略

- 黄金坩埚阶段，Header 右上项目 / 文稿选择器已置灰不可点击。
- 当前 hover 文案已转为“议题尚未定稿，暂不可创建项目 / 文稿”。

### 5. 议题标题收敛首版

- 已把右侧 chat 输出与中区标题做基础联动。
- 当前会从最近一轮 assistant 输出中提取可用标题，替换掉“标题待收敛...”。
- 该逻辑仍属首版，后续需继续收紧误判。

### 6. 滚动 bug 修复

- 用户在填写问题 1 并点击保存后，中区主滚动条出现失效。
- 本轮已修复：
  - 保存后主动让当前 `textarea` 失焦
  - 将问题卡输入框改为固定高度 + 内部滚动
- 目标是避免 `textarea` 撑开容器或焦点锁定导致中区滚动体验异常。
