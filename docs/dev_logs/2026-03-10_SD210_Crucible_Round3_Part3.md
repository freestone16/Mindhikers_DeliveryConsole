## 本轮目标

围绕第三轮“议题锁定阶段”继续收口三个现场问题：

1. 中区默认不应预灌内容，必须先留白
2. 第一轮与后续轮次应由 Socrates 驱动，而不是前端机械硬编码
3. 每轮提交后，右侧应有老张 / 老卢读完答案后的真人式反馈，而不是只弹系统状态

## 已完成

### 1. 中区默认空场成立

- 已移除黄金坩埚工作台对 `INITIAL_ASSETS` 的默认依赖。
- 页面初始态改为真正留白：
  - 左侧结晶目录只显示空态说明
  - 中区显示黄金坩埚留白画面
  - 只有在右侧先抛出命题后，中区才开始生成问题与资产

### 2. 第一轮改为“由第一次提问触发”

- 首轮问题不再在页面加载时自动长出。
- 右侧用户第一次发问后，宿主层会记录 seed prompt，并触发 Socrates 生成第一轮问题。
- `reset` 现在会同时清空：
  - 中区问题卡
  - 宿主层 routed assets
  - 右侧 crucible chat 历史
  - seed prompt 状态

### 3. Socrates 返回结构升级

- `server/crucible.ts`
- Socrates 生成接口不再只返回 3 张问题卡，而是返回：
  - `speaker`
  - `reflection`
  - `cards`
- 含义是：
  - 谁在继续追问
  - 他看到了你上一轮回答里的什么
  - 他接下来给出的三张问题卡是什么

### 4. 后续轮次具备“读完回答再追问”的互动感

- 第二轮及之后的问题生成完成时，右侧 chat 会额外注入一条老张或老卢消息。
- 这条消息不再是“已送资产区 / 已生成问题”这类状态句，而是：
  - 先概括用户上一轮回答中已暴露出来的东西
  - 再引出下一轮追问方向
- 第一轮仍保留正常 chat 回复，不额外叠加 injected message，避免第一问就双回。

### 5. 资产分流规则继续收紧

- `src/components/crucible/hostRouting.ts`
- 不再因为一句话里出现“资产区 / 结构 / 画面”就机械送中区。
- 当前只有更明显的结构化内容、参考材料或图谱型内容才会被送入中区。
- 目标是减少“短状态句误判成资产”的噪声。

### 6. 右侧重复发送增加首版防抖

- `src/components/ChatPanel.tsx`
- 当前已补：
  - 发送中的硬锁
  - 同内容短时间重复发送抑制
- 这是针对用户现场看到的“同一句被发两次 / 回两次”的第一层兜底。
- 该问题仍需用户现场继续回归确认。

### 7. UI 小问题收口

- 问题卡右上“已保存 / 未保存”徽标统一为圆角 pill，不再因为宽度压缩出现一方一圆。
- 用户侧不再暴露“本地兜底问题集”这种开发词。
- 若 Socrates 当下未成功返回，现在页面只显示“先用临时追问继续”，避免打断用户流程。
