### P1：Chatbox 改类型后前端不刷新（已修复）

**调查过程**：
1. 用 Socket.IO 客户端模拟 `chat-action-execute`，确认后端数据流完整：
   - `expert-data-update:Director` 正确广播，ch1 types 已更新
   - `chat-action-result.expertState` 正确携带完整状态
2. 用 preview 工具在浏览器内模拟 action，确认 `[useExpertState] 🎯 chat-action-result carrying fresh state` 日志触发
3. 截图确认 UI 正确更新（1-1 显示"互联网素材"绿色标签 + 上传入口）

**根因（双重）**：
- **工具定义错误**：`update_option_fields` 工具描述中的 type 可选值使用了错误名称（`remotion-animation`、`text-to-video`、`screen-recording` 等），与系统实际值不符（`remotion`、`seedance`、`artlist`、`internet-clip`、`user-capture`、`infographic`）。LLM 发送了错误 type 值，保存后前端 TYPE_LABELS 找不到匹配项。
- **React key 不含 type**：OptionRow key 仅包含 `imagePrompt|props`，type 变更时若 props 未变则组件不 remount，视觉层无法保证刷新。

**修复**：
- `server/expert-actions/director.ts:64` — 修正工具定义中 type 可选值
- `src/components/director/ChapterCard.tsx:462` — key 中加入 `option.type`
- `server/index.ts:982` — 添加 `📦 expertState attached` 诊断日志
