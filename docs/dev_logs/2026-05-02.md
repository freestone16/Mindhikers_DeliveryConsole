## Unit 5 首段：Runtime Action Trace Foundation

### 本轮目标

按 `docs/plans/2026-05-01-director-design-system-ui-implementation-plan.md` 推进 Unit 5：状态透明与审计链路。执行口径是轻量打底，不新增持久审计 store，先让已有 Director 主动作进入统一 runtime 数据链路。

### 代码变更

#### `src/types.ts`

- 新增 `RuntimeActionEvent`、`RuntimeActionType`、`RuntimeActionStatus`、`RuntimeData`。
- `RuntimeData` 统一承载当前模型、日志、结构化动作、当前 pending 动作、loading/startTime 与 phase。

#### `src/components/DirectorSection.tsx`

- 新增 `runtimeActions` 与 `pushRuntimeAction()`。
- P1 生成、P1 修订、P1 批准、P2 生成、P2 选择/确认/批量确认、P3 批准/修订/批量批准、P2→P3、P3→P4、导入章节、阶段切换，都会写入结构化 action trace。
- `onRuntimeDataChange` 现在向 shell 传递 `actions`、`activeAction` 与当前 phase。
- P1 生成/修订也接入 `startTime`，底部状态栏能看到当前动作。

#### `src/components/director/Phase2View.tsx`

- 新增可选 `onRuntimeAction`，向下传给 `ChapterCard`。

#### `src/components/director/ChapterCard.tsx`

- P2 单卡片的素材上传、上传失败、预览生成、预览任务提交、预览处理成功/失败，都进入 runtime action trace。
- 失败重试会标记为 `retry` 类型。

#### Shell / Drawer

- `src/App.tsx`、`DeliveryShellLayout.tsx`、`ContextDrawer.tsx` 改用共享 `RuntimeData` 类型。
- `RuntimePanel` 优先展示结构化 `actions`，没有 actions 时才回退用日志关键词。
- `DeliveryStatusBar` 显示当前 pending 动作；空闲时显示最近动作结果。

#### 测试

- 新增 `src/components/delivery-shell/drawer/RuntimePanel.test.tsx`：
  - 验证结构化 action trace 可见。
  - 验证 pending active action 会显示为当前动作。
