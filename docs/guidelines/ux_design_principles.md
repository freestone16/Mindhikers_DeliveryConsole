# MindHikers Delivery Console — UX 设计原则

> 本文件为全项目级 UX 设计决策的强制参照标准。
> 任何涉及 UI/交互方案设计时，研发人员和 AI 助手都必须回顾本文件。

## 原则 1：沿用知名软件逻辑 (Familiar UX Patterns)

> 以**一般自媒体创作者**的视角分析考虑，优先采用用户已熟悉的交互范式，降低学习心智成本。

**核心理念**：我们的用户不是程序员，而是自媒体创作者。他们day-to-day最频繁使用的工具是 Excel、剪映、YouTube Studio、Notion 等。UI 交互应尽量向这些工具看齐。

**参照映射表**：

| 交互场景      | 参照软件       | 具体范式                    |
| ------------- | -------------- | --------------------------- |
| 批量选择/全选 | Excel          | 表头三态 checkbox (☐/☑/[-]) |
| 列筛选/过滤   | Excel          | 下拉筛选菜单，"全选"按钮    |
| 拖拽排序      | 剪映/Figma     | 拖拽手柄 + 实时预览         |
| 状态流转      | YouTube Studio | 草稿 → 审核 → 发布          |
| 快捷键        | VS Code        | Cmd+S 保存, Cmd+Z 撤销      |

**禁止行为**：
- ❌ 发明全新的交互范式（除非有充分理由并经老卢确认）
- ❌ 以技术实现方便为由牺牲用户体验
- ❌ 循环调用 toggle 代替一次性批量操作

## 原则 2：数字一致性 (Single Source of Truth for Counts)

所有 UI 计数必须从同一个 filter/计算函数取数，严禁多处独立计算导致数字矛盾。

## 原则 3：操作原子性 (Atomic Batch Operations)

批量操作必须一次 setState 完成。严禁在循环中逐个调用状态更新函数（会导致竞态条件，只有最后一个写入存活）。

## 原则 4：渐进式信息披露 (Progressive Disclosure)

- 默认展示核心信息，高级选项折叠
- 调试面板默认收起
- 错误信息先 toast 摘要，详情按需展开
