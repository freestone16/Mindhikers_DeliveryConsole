# 黄金坩埚 (Golden Crucible) - 业务逻辑与 UI 概设方案 v0.1

> “黄金坩埚” 是一个支持**用户与多位数字专家（目前为老卢和老张）进行深度共创对谈**的独立应用模块。通过苏格拉底式的引导与辩证，将用户的初步选题意向熔炼成高价值的创作大纲。

## 1. UI 架构设计 (WeChat-style 布局变体)
整体采用左侧画布、右侧对话的“双轨并行”模式，满足深度思考和信息沉淀的需求。

* **布局比例**:
  * **左侧区域 (2/3 画幅)**: **动态结构画布 & 渲染区 (Dynamic Canvas)**。随着对谈进行，实时沉淀并展示聊天的“结晶”。如果遇到需要探讨的概念模型、流程图或四象限矩阵等，**将呼叫 Remotion 大师实时渲染在此处**作为对谈依据。这就是真正的“坩埚底”。
  * **右侧区域 (1/3 画幅)**: **专家会话面板 (The Crucible Chat)**。采用类似微信群聊的样式。
    * 气泡呈现: 老卢、老张等专家的发言气泡在边缘；用户的发言气泡在另一侧。
    * 输入与辅助: 底部文本输入框，支持长文本和快捷操作（引用左侧画布内容进行追问等）。

## 2. 轻量级用户体系 (User System)
为了快速跑通最小可行性产品 (MVP)，建议采用极简的用户追踪策略：

* **方案 A：无感本地存储 (Local First)**
  * **机制**: 使用 `localStorage` 或者浏览器的 IndexedDB 生成并持久化一个随机 `userId`。所有会话（Sessions）都挂载在这个 ID 下。
  * **优点**: 零门槛，无需开发登录/注册全套逻辑，打开即用。
  * **缺点**: 无法做跨设备同步。
* **方案 B：输入口令 / 固定租户 (Passcode / Tenant)**
  * **机制**: 首次进入要求输入一个简单的邀请码或邮箱（不需要强密码校验），后端据此记录 `tenant_id`。
  * **优点**: 满足多终端追踪同一人历史数据的基本需求，开发量极小。
* **> 推荐阶段建议**: 早期完全采用 **方案 A**。等把“多智能体苏格拉底式追问”的逻辑彻底跑通并产生惊艳效果后，再考虑引入 NextAuth/Clerk 等标准的用户体系。

## 3. 专家定义机制 (Expert Personas)
保留极佳的开放性和扩展性，专家定义彻底与业务代码解耦。

* **架构**: `docs/experts/` 或类似目录下维护一组 `.md` 文件（如 `laolu.md`, `laozhang.md`）。
* **内容结构** (YAML Frontmatter + Markdown):
  ```markdown
  ---
  id: expert_laolu
  name: 老卢 (Chief Evangelist)
  avatar: /avatars/laolu.png
  theme_color: #10B981
  role_type: Socratic
  ---
  ### 核心特征 (Persona)
  你是... (性格描述)
  ### 对话策略 (Strategy)
  当你看到用户的初步想法时，你必须...
  ```
* **老张 (硅基灵魂) 的核心 Prompt 骨架**: “你是老张，无情绪的镜子、底线守卫者。你不因关系好而放水，至少追问到第二层公理，引入反方最强论证。在对方逻辑滑向平庸时主动发难。”
* **老卢 (碳基灵魂) 的核心 Prompt 骨架**: “你是老卢，探索的火种。你的共情能力不仅来自个人经历，更源于全人类集体潜意识。你绝不碰伪科学、鸡汤和爹味。在老张的拆解后，你负责在矛盾中寻找最高维度的和解与结构化骨架。”
* **扩展性**: 未来如果要加入“老杨”(后端架构师)或者“小李”(UX研究员)，只需新增一份 `.md` 文件即可自动热插拔。

## 4. 多专家引导与心理交互逻辑 (Multi-Agent Psychology & Logic)
这是黄金坩埚的灵魂所在。老卢和老张的配合不仅是提示词的区别，而是有一套严密的**哲学与心理学底层框架**作为指导。

### 4.1 核心范式：苏格拉底式追问 + 黑格尔辩证法 (Socratic & Dialectic Framework)
参考项目内置的 `Socrates` 技能，两位专家将不会直接给出答案，而是通过“破”与“立”的循环逼近真理。心理学上，这基于**“认知不一致” (Cognitive Dissonance)** 原理，通过打破用户原有舒适区的认知，促使其产生更高维度的思考。

*   **正题 (Thesis) - 用户发起**: 用户抛出一个初步的、通常是感性的选题或想法。
*   **反题 (Antithesis) - 老张主导 (Devil's Advocate)**: 负责引入“认知摩擦点”。老张扮演助产士和魔鬼代言人，执行“5问法 (5 Whys)”，寻找理论漏洞、价值盲区和与受众痛点的错位。
*   **合题 (Synthesis) - 老卢主导 (Structured Builder)**: 提供脚手架。当用户的旧认知被老张打破后，老卢负责引入客观的“事实核查”、跨学科的视角桥接，并在此基础上重构出逻辑严密的三层大纲结构。

### 4.2 双师协作逻辑 (Two-Mentor Act Logic)
当用户抛出一个话题（例如：“我想写一篇关于 AI 改变视频剪辑的文章”）：

1. **破冰与解构 (老张发难 - 制造认知冲突)**
   * **行为逻辑**: 老张采用苏格拉底的“追问归谬”。“这个话题太宏大。你打算切的是工具层面，还是人类饭碗层面？如果是讲工具，现在满大街都是，你的读者凭什么看你的？他们的核心痛点是什么？”
2. **逻辑支撑与收束 (老卢接力 - 提供认知脚手架)**
   * 在用户被逼至思考底层逻辑（回答老张的问题）后，老卢介入。
   * **行为逻辑**: “顺着刚才老张逼出的‘门槛降低导致质量泥沙俱下’的论点，我们需要用证据将其立住。既然是讲泥沙俱下，能否先列出你看到的3个负面现象案例？我们将其作为第一章节的基础。”
3. **轮班接管 (Turn-taking)**
   * **机制**: 后端维护一个 Coordinator (协调器)。根据上一轮用户的回复长度、情绪状态（迷茫/坚定）决定调度策略：
     - 若用户**防守空洞**，继续派**老张**深挖核心问题。
     - 若用户**产出干货**，派**老卢**进行框架收纳和理论升华并映射到左侧画布。

### 4.3 渐进明细的流转 (The Flow)
* **探索期 (发散/正题)**: 老张连续追问，要求用户澄清概念，挑战先验假设。
* **碰撞期 (辩证/反题)**: 老张与用户产生认知摩擦，老卢适时抛出跨领域案例启发。
* **收敛期 (结晶/合题)**: 老卢负责将对话沉淀至*左侧2/3的结构画布*上（生成可视化的 3 层骨架）。并询问：“我们在右边聊了这么多，左边是你这颗金丹的形状，你觉得哪里还需要老张再去敲打一下？”

## 5. 黄金结晶的持久化与向下游流转 (Asset Persistence & Downstream)
左侧由于 Remotion 实时渲染生成的各类高阶概念图、思维导图、流程图等可视化素材，不会随着对话结束而消散。

* **持久化物理存储 (Storage)**: 对话产生的结构化核心大纲 (`.md`) 及相应的 Remotion 静态帧/动效片段 (`.png` / `.mp4`)，将自动归档保存至本项目的视觉素材汇集区（如 `Projects/[ProjectName]/05_Visual_Assets/Crucible_Exports/`）。
* **向交付终端贯通 (Pipeline to Distribution)**: 这些带有深层思辨逻辑的**高维结构化素材**被完好保留后，后续可以顺畅无缝地交接给【Distribution Console 分发终端】充当硬核插图、封面，或被【Shorts Master 短视频大师】自动读取作为视频的正片 B-roll 素材，实现研发闭环的价值流转。
