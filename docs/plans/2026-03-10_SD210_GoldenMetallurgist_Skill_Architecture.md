## 1. 本轮定稿目标

本轮不是继续扩 UI，也不是继续把后续交付终端拉进来，而是先把黄金坩埚当前阶段的逻辑边界定清。

当前阶段的目标被收窄为：

1. 完成议题锁定
2. 完成深度对话与结构收束
3. 最终产出可立住的论文级文本

当前阶段的终点先定义为：

- `ThesisWriter` 论文产出

以下暂不纳入本阶段主目标：

- 交付终端接线
- `Writer` 视频文案生成
- 用论文继续生产视频脚本

## 2. 前台与后台的原则

### 2.1 前台只出现三者

黄金坩埚前台只允许稳定出现：

- 用户
- 老张
- 老卢

其中：

- 老张：硅基数字人，负责拆概念、施压、引入反方
- 老卢：碳基数字人，代表用户本人精神内核的数字化，负责立结构、托举、重构

### 2.2 黄金冶炼师默认隐身

黄金冶炼师不作为常规前台角色出现。

它的存在方式是：

- 后台导演
- 后台调度器
- 阶段控制器

默认不在对话中显名出现，最多作为极少数彩蛋或系统级裁决露面。

### 2.3 工具不是人

以下都不是前台人格，而只是后台工具：

- `Socrates`
- `Researcher`
- `FactChecker`
- `ThesisWriter`

它们不应以“角色”形式出现在用户视野里。

## 3. 老张 / 老卢的 SSOT

老张、老卢不做 skill。

它们的单一事实源应是：

- `docs/02_design/crucible/souls/oldzhang_soul.md`
- `docs/02_design/crucible/souls/oldlu_soul.md`

当前确认：

- 两份 soul 文档仍在主现场，未被冲掉
- 当前运行链路尚未真正加载它们
- 这意味着人格设计仍停留在文档层，后续需要接线到运行时

## 4. Socrates 的正确定位

`Socrates` 不是角色，而是“追问方法”。

它的作用是：

- 逼概念边界
- 暴露前提
- 追问反例
- 迫使观点从感受走向判断

它不应在前台显名。

用户不应该看到：

- “苏格拉底说”
- “Socrates 建议”

正确方式是：

- 黄金冶炼师在后台调用 `Socrates`
- 再把产出转译为老张 / 老卢的发言，或中区问题卡

## 5. 黄金冶炼师的职责

黄金冶炼师应被定义为一个新的 skill，但它不是第四个数字人，而是调度 skill。

它只负责：

1. 判断当前阶段
   - 议题锁定
   - 深度对话
   - 结晶收束
2. 判断当前需要哪种方法
   - 追问
   - 拆概念
   - 立结构
   - 搜索
   - 校验
   - 论文收束
3. 决定由谁对用户说
   - 老张
   - 老卢
   - 或中区问题卡
4. 决定何时进入下一阶段

黄金冶炼师不负责：

- 作为常驻角色和用户聊天
- 替代老张 / 老卢人格
- 直接管理 UI
- 直接处理项目创建逻辑

## 6. 当前阶段的工具调用链

当前阶段建议的逻辑链如下：

1. 用户输入议题
2. 黄金冶炼师判断当前是否需要追问
3. 若需要追问，后台调用 `Socrates`
4. 根据调度结果，将内容转译为：
   - 老张发言
   - 老卢发言
   - 或中区问题卡
5. 若需要真实世界支撑，再调用：
   - `Researcher`
   - `FactChecker`
6. 当议题足够收束时，调用：
   - `ThesisWriter`

当前阶段的最终产物：

- 论文级输出

## 7. ThesisWriter 的位置

本轮定稿补充：

- `ThesisWriter` 才是当前阶段的最终写作者
- 它负责将坩埚沉淀出的议题、结构、判断、金句与参考材料整理为论文级文本
- 在本阶段中，`Writer` 暂不作为主链角色讨论

换句话说：

- 黄金坩埚当前阶段 = 从议题到论文
- 不是从议题直接到视频文案

## 8. 当前阶段的最终定义

当前黄金坩埚的产品本质不是“一个带聊天框的协作工具”，而是：

- 用户与两个数字人对话
- 一个隐身导演在后台调度方法
- 通过追问、拆解、重构、校验与收束
- 最终产出论文级文本
