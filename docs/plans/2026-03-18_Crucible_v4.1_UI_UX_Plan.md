# 黄金坩埚 v4.1 UI/UX 优化方案

> 日期：2026-03-18
> 基线版本：v4.0.0 (`0fee43f`)
> 分支：`codex/crucible-main`（远端 `MHSDC-GC`）
> 作者：老杨 + 老卢

## Context

v4.0.0 完成了 prompt 架构重写（删除 19+14 条行为规则，注入 Socrates v0.2.0 全量 + Soul 文档），对话质量已验证可用，老张老卢风格区别明显。

本轮目标：**前端交互体验打磨**，不动 prompt 架构。

## 改动清单（8 项，按优先级排序）

### 1. 标题自动收敛 + 去重
**文件**: `src/components/crucible/CrucibleWorkspaceView.tsx`, `server/crucible-orchestrator.ts`, `server/crucible.ts`

- **前几轮不动标题**：focus 是追问方向不是命题本身，前 1-2 轮保持"标题待定"
- **第 3 轮起收敛标题**：在 LLM 输出 JSON 里新增 `topicSuggestion` 字段，仅在 `deep_dialogue` 阶段（`roundIndex >= 3`）要求 LLM 生成
- 后端 `buildSocratesPrompt` 在 roundIndex >= 3 时，输出 schema 加一行：`"topicSuggestion": "可选，当讨论已有足够上下文时，凝练一个短标题"`
- 后端 `crucible.ts` 解析 `parsed.topicSuggestion`，通过 API 返回给前端
- 中屏标题区：收到 `topicSuggestion` 后替换"标题待定"
- 右侧 ChatPanel header：去掉重复的标题显示，改为固定文字"对话"或直接隐藏 displayName
- `normalizeTopic()` 保留作为 fallback

### 2. 左侧结晶区激活
**文件**: `CrucibleWorkspaceView.tsx`

- 当前左侧只显示 presentables 的索引按钮，结晶内容（核心判断、金句）没有被沉淀
- 方案：每轮对话后，自动从 `presentables` 中提取 `type=quote` 的条目，累积到"结晶"列表
- 结晶区显示格式：简短金句 + 轮次标记，点击可跳回对应黑板内容

### 3. 空间优化
**文件**: `CrucibleWorkspaceView.tsx`, `ChatPanel.tsx`, `App.tsx`

| 项目 | 当前值 | 目标值 |
|---|---|---|
| 全局圆角 | 24px / 22px / 18px | 12px / 10px / 8px |
| 中屏与 chatbox 间 gap | `gap-3`(12px) + `pr-5`(20px) | `gap-2`(8px) + `pr-2`(8px) |
| 中屏竖线分割位置 | chatbox `min-width: 420px` | `min-width: 380px`，让中屏更宽 |
| 左侧结晶宽度 | 172px | 保持不变 |

### 4. 中屏与 chatbox 视觉区分
**文件**: `CrucibleWorkspaceView.tsx`, `ChatPanel.tsx`

- 中屏黑板：字体放大 `text-[15px] leading-8`，更像板书/笔记（保持文字密度，看效果再调）
- chatbox 对话区：保持 `text-[13px] leading-7`
- 中屏标题用更重的 serif 字重，chatbox 用 sans-serif

### 5. Header 加"心行者"
**文件**: `src/components/Header.tsx` (Line ~107)

- `MindHikers Delivery Console` → `心行者 MindHikers Delivery Console`

### 6. Chatbox 增强（深度思考友好）
**文件**: `ChatPanel.tsx`

- **保存草稿按钮**: 输入区加"保存"按钮，将当前输入文本存到 localStorage 草稿
- **发送确认**: 可配置项 `confirmBeforeSend`，开启后发送前弹出 y/n 确认
- **配置项存储**: `useState` + localStorage 持久化
- 这两项做成 ChatPanel header 右上角小齿轮里的开关（可配置，因为有人需要有人不需要）

### 7. "思"字和圆标去掉
**文件**: `ChatPanel.tsx` (Line ~833)

- 删掉 `<div>思</div>` 的 9x9 avatar 方块
- 保留"推理中"/"老卢、老张在思索"文字作为系统提示
- 改为更简洁的行内提示样式（无 avatar，无圆标）

### 8. 对话导入/导出功能
**文件**: `ChatPanel.tsx`, `src/components/crucible/storage.ts`

- 当前已有：导出 Markdown（Download 按钮）
- 新增"导入"按钮（Upload icon），放在下载按钮旁边
- 导入格式：JSON（直接恢复 `CrucibleSnapshot`）
- 导出新增 JSON 格式选项（除现有 Markdown 外）
- 导入后恢复完整对话状态，可以接着聊
