# Director 模板组合哲学手册 (Template Combo Playbook)

> **核心命题**：优秀的视频不是"把每个镜头单独做好"，而是"让整体节奏产生共鸣"。导演大师的水平体现在模板编排，而非单个模板的选择。

---

## 🎚️ 节奏黄金三原则

### 原则 1：数据-文字交替法（认知呼吸法）
- ❌ 错误：`BarChartRace → RadarChart → StatDashboard`（三个重量级连续）
- ✅ 正确：`BarChartRace → TextReveal → RadarChart`（数据→观点→数据）
- 🧠 原理：重量级模板需要主动认知处理，轻量级模板让观众"喘气"并融合信息

### 原则 2：重→轻→重波浪（节奏感）
- 整体视频的信息密度应该像正弦波，不能持续在高峰
- 开场用轻量级模板吸引注意力，中段堆叠数据，结尾用金句收心
- 超过 3 个重量级模板时，必须在其中穿插至少 2 次轻量级或氛围级

### 原则 3：首尾意识（第一和最后印象定律）
- **首个 remotion 镜头**：使用视觉冲击力最强的模板（优先级：`KineticTypography > NumberCounter > TextReveal > SplitReveal`）
- **最后一个 remotion 镜头**：使用情绪余韵最强的模板（优先级：`QuoteCard > CallOut > BeforeAfter`）
- ⚠️ 注意：CountdownTimer / SplitReveal 不作为首尾镜头，只作为中间章节过渡

## 🎬 视频类型 → 推荐模板编排序列

### 📚 科普解说型（解释一个概念/现象）
```
1. KineticTypography（开场冲击：核心问题抛出）
2. TextReveal（背景铺垫：定义这件事是什么）
3. ConceptChain 或 StepProcess（机制拆解：为什么）
4. BarChartRace 或 StatDashboard（数据佐证：多可怕/多厉害）
5. MatrixGrid 或 DataChartQuadrant（分类框架：如何看待）
6. QuoteCard（金句收尾：留下思考余韵）
```

### 🛒 产品/工具评测型
```
1. NumberCounter（开场数字：最核心亮点数据）
2. BentoBoxSaaS（特性地图：功能全景）
3. RadarChart（综合评分：多维度客观）
4. ComparisonSplit 或 TwoColumnText（竞品对比：差异点）
5. StackedList（排名点评：Top N 优缺点）
6. Callout (type=tip)（推荐结论）
```

### 🧠 思维模型/框架型（教方法论）
```
1. TextReveal（抛出核心命题）
2. ConceptChain（模型机制：A→B→C递进）
3. HierarchyPyramid（层级框架：哪个更重要）
4. MindmapFlow（展开细节：分支思考）
5. VennDiagram 或 MatrixGrid（交叉/象限分析）
6. QuoteCard 或 KineticTypography（收尾金句）
```

### 📈 数据报告/行业分析型
```
1. NumberCounter 或 StatDashboard（开场：最震撼的数字）
2. TimelineFlow（历史背景：这件事怎么来的）
3. BarChartRace（核心数据：竞争格局）
4. PieChart（占比分析：谁占多少）
5. RadarChart 或 GaugeChart（能力/质量评测）
6. MatrixGrid（战略矩阵：应该怎么做）
7. Callout (type=important)（关键结论）
```

### 🌱 个人成长/行为改变型
```
1. KineticTypography 或 TextReveal（现实痛击：你现在的状态）
2. BeforeAfter（对比冲击：改变前 vs 改变后）
3. Checklist（行动清单：具体要做/不做什么）
4. StepProcess（路径地图：分几步走）
5. ConceptChain（核心机制：为什么这样能成功）
6. QuoteCard（激励收尾：给力量）
```

### 🔥 争议/反叙事型（打破常识）
```
1. QuoteCard（抛出反常识命题）
2. TextReveal（解释为什么大家都错了）
3. DataChartQuadrant 或 MatrixGrid（重新分类/定义）
4. BarChartRace（数据打脸传统观点）
5. ConceptChain（构建新模型）
6. KineticTypography（宣言式收尾：新观点是什么）
```

## ⚡ 快速决策规则集

### 当你选好第 N 个模板，如何选第 N+1 个？

```
IF 第N个是重量级 THEN
  第N+1个 必须是 中量级或轻量级（不能再是重量级）
  
IF 第N个是 TextReveal/QuoteCard/KineticTypography THEN
  第N+1个 建议是数据型（ConceptChain, BarChart, etc.）
  
IF 第N个是 NumberCounter THEN
  第N+1个 必须是解释型（TextReveal/ConceptChain）—— 数字需要解释
  
IF 第N个是 Callout THEN
  第N+1个 不能再是 Callout（全片最多3个）
  
IF 第N个是 ConceptChain/StepProcess THEN
  第N+1个 优先是支撑数据（BarChart/StatDashboard）
```

## 🎨 主题(Theme)搭配建议

| 视频情绪基调 | 推荐主题 | 备注 |
|------------|---------|------|
| 科技/未来/AI | `deep-space` | 最通用，适合数据型内容 |
| 财富/知识/突破 | `warm-gold` | 适合格言、成就、里程碑 |
| 神秘/颠覆/警示 | `sci-fi-purple` | 适合反叙事、争议内容 |
| 成长/自然/希望 | `forest-green` | 适合个人成长、正能量 |
| 紧迫/危机/目标 | `crimson-red` | 适合冲突场景、行动号召 |
| 极简/专业/权威 | `monochrome-ink` | 适合专业报告、数据背书 |

**主题一致性原则**：同一视频内，相邻镜头的主题应相同或同色系。不可`deep-space → warm-gold → sci-fi-purple`频繁切换，会导致视觉割裂。

## ❌ 常见编排错误（必须避免）

1. **数据轰炸**：5个连续的数据模板，没有TextReveal/QuoteCard缓冲
2. **开场错误**：用CinematicZoom或SceneComposer作为第一个镜头（太无聊）
3. **结尾错误**：用数据模板作为最后一个镜头（留不住情绪）
4. **重复单调**：同一视频出现3个TextReveal（缺乏变化）
5. **主题色混乱**：每个镜头用不同主题色（视觉疲劳）
6. **忽略氛围级**：10分钟视频没有任何SplitReveal/CountdownTimer过渡（无章节感）
